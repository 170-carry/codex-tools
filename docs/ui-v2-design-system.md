# Codex Tools 2.0 UI 设计规范

## 设计方向

2.0 UI 改成工具台风格，核心目标是让账号状态、额度风险和操作入口一眼可见。整体不再使用旧版暖色玻璃卡片和圆环进度，改为白底、深墨文字、蓝色主操作、绿色/琥珀/红色语义状态。

## 首页结构

1. 顶部区域：品牌、分段导航、主题切换、刷新。
2. 指标区：账号总数、活跃账号、即将耗尽、异常账号。
3. 工具条：搜索账号/邮箱、状态筛选、套餐筛选、反代筛选。
4. 主列表：账号身份、套餐标签、状态、5 小时使用率、周使用率、重置时间、反代开关、切换操作。
5. 右侧详情栏：账号身份、使用概览、重置时间、套餐类型、更新时间、账号来源、最近记录、快捷操作。

首页外层内容框必须贴合应用窗口，不再保留大面积内边距。macOS 原生窗口按钮由系统提供，页面内不再重复绘制红黄绿按钮。品牌图标使用 2.0 工具台图标。

## 色彩系统

| 类型 | 颜色 | 用途 |
| --- | --- | --- |
| Primary | `#2563EB` | 主按钮、选中边框、主进度 |
| Text | `#0F172A` | 标题和主要内容 |
| Muted | `#64748B` | 次级文字、说明文字 |
| Success | `#22C55E` | 可用、使用中、正常状态 |
| Warning | `#F59E0B` | 即将耗尽、额度风险 |
| Danger | `#EF4444` | 异常、删除、错误 |
| Background | `#F6F8FB` | 页面底色 |
| Card | `#FFFFFF` | 面板、列表行、详情卡片 |

## 核心组件

### 分段导航

用于一级页面切换。选中态使用白色胶囊底和蓝色文字，未选中态只保留深灰文字。

### 指标卡

指标卡固定为图标、标题、数字、说明四部分。颜色只表达语义，不额外堆叠装饰。

### 筛选控件

搜索框和下拉筛选高度统一为 42 到 44px。筛选只做当前页面状态过滤，不改变账号原始数据。

### 账号列表行

列表行是首页主组件。每行高度稳定，左侧展示头像、账号名、套餐标签和状态；中间展示线性进度；右侧放反代开关、切换按钮和导出按钮。列表行不使用固定宽度表格，必须在常见窗口宽度下完整展示操作列。选中态只强调边框和左侧蓝色线。

### 进度条

进度条统一展示使用率。正常为蓝色，剩余低于 35% 改为琥珀色，剩余低于 15% 改为红色。

### 状态标签

状态分为使用中、可用、即将耗尽、异常账号。状态点颜色必须和语义色一致。

### 详情栏

详情栏用于承载当前选中账号的二级信息。结构为身份头、使用概览、元信息、最近记录、快捷操作。详情栏不再重复大面积装饰，只通过边框和分区保持可读性。

### 添加账号弹窗

添加账号弹窗属于组件级规范的一部分。导入方式使用 2x2 卡片网格，图标容器统一为 52px 描边方块，选中态只使用蓝色边框和浅蓝底。下方内容面板使用同样的白底细边框，输入框、状态块和上传队列不再使用旧版渐变背景。

### 快捷操作

快捷操作使用图标加文字。危险操作必须使用红色语义态，并和普通操作保持视觉区分。

## 交互规则

1. 点击列表行会切换右侧详情栏。
2. 点击套餐标签会在同一账号的多个变体之间切换。
3. 搜索和筛选只影响左侧列表，右侧会自动跟随当前可见账号。
4. 删除按钮继续沿用二次确认机制。
5. 别名编辑只在详情栏完成，避免列表行被输入控件挤压。

## 响应式规则

宽窗口使用左列表右详情。窄窗口下详情栏移动到列表下方，账号行改为上下分区排布，避免隐藏反代、切换和导出操作。移动宽度下指标卡变成单列，快捷操作从四列降为两列或一列。
